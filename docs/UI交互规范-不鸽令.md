# UI/交互规范 - 不鸽令·线下契约引擎

---

## 1. 设计原则

- **极简优先：** 每个页面只做一件事，减少认知负担
- **信任感：** 通过金额展示、信用分、契约声明建立用户信任
- **高对比度：** 关键信息（金额、状态、倒计时）必须一眼可见

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色 | `#FF6B35` | 亮橙色，用于按钮、重点标签 |
| 辅助色 | `#1A1A2E` | 深蓝黑，用于标题文字 |
| 成功色 | `#10B981` | 绿色，核销成功、信用良好 |
| 警告色 | `#F59E0B` | 黄色，倒计时、待处理状态 |
| 危险色 | `#EF4444` | 红色，违约、封号、举报 |
| 背景色 | `#F5F5F7` | 浅灰，页面背景 |
| 卡片背景 | `#FFFFFF` | 白色 |
| 次要文字 | `#6B7280` | 灰色，辅助信息 |

## 6. 状态标签样式

| 状态 | 背景色 | 文字色 | 文案 |
|------|--------|--------|------|
| pending | `#FEF3C7` | `#D97706` | 待组队 |
| confirmed | `#DBEAFE` | `#2563EB` | 已成行 |
| verified | `#D1FAE5` | `#059669` | 已核销 |
| expired | `#FEE2E2` | `#DC2626` | 已超时 |
| settled | `#E5E7EB` | `#6B7280` | 已结算 |
| breached | `#FEE2E2` | `#DC2626` | 违约 |

## 7. 动效规范

| 场景 | 动效 | 时长 |
|------|------|------|
| 页面切换 | 右滑进入/左滑返回 | 300ms |
| 核销成功 | 绿色对勾放大弹出 | 500ms |
| 支付成功 | 卡片翻转显示核销码 | 400ms |
| 倒计时刷新 | 二维码淡入淡出切换 | 200ms |
| 按钮点击 | 缩放 0.95 → 1.0 | 150ms |
| Toast 提示 | 底部滑入 | 250ms |

## 8. 空状态设计

| 场景 | 插图描述 | 文案 |
|------|----------|------|
| 附近无活动 | 空地图 + 放大镜 | "附近暂无活动，去发起一个？" |

| 网络异常 | 断开的信号 | "网络不给力，点击重试" |
